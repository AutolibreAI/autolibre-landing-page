# Implementation Plan: AutoLibreAI Landing Page — Next.js Best Practices

**Branch**: `002-autolibre-landing` | **Date**: 2026-05-12 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/002-autolibre-landing/spec.md`

## Summary

Migrate the existing AutoLibreAI landing page to strict Next.js best practices: fix all lint/type warnings, replace raw `<img>` elements with `next/image`, extract the inline nav to its own component, restore the `eyebrow` rendering in `BrandHeading`, remove the color gap above the problem section, and apply correct HTML language metadata. Zero new runtime dependencies required.

**Language/Version**: TypeScript 5, strict mode

**Storage**: N/A (static landing page)  
**Testing**: Manual visual QA + `npm run lint` + `tsc --noEmit`  
**Target Platform**: Web — desktop and mobile browsers  
**Project Type**: Static marketing landing page (Next.js App Router, Server Components)  
**Performance Goals**: Lighthouse performance ≥ 90 on mobile  
**Constraints**: Zero `// eslint-disable` or `// @ts-ignore`; no new runtime dependencies  
**Scale/Scope**: Single page, ~10 components
